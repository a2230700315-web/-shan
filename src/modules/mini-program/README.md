# mini-program 模块说明

本目录用于在当前 React Web 仓库中沉淀“小程序端可执行前端契约”，供后续微信小程序工程直接复用。

## 目录结构

- `types.ts`：核心类型定义（角色、节点、页面、订单、消息模板）
- `constants.ts`：信息架构、页面清单、节点流、角色权限矩阵、消息模板
- `fieldDictionary.ts`：字段字典（发布需求表单、各节点操作字段规范）
- `flow.ts`：流程辅助函数（权限判断、节点推进、超时规则、议价轮次）
- `mock.ts`：订单与议价流程模拟数据
- `tests.ts`：契约完整性与关键规则校验
- `index.ts`：统一导出入口

## 已覆盖需求（对应 V3.1）

1. 小程序双门户：用户端 + 服务商端
2. 服务商角色矩阵：管理员/客服/地勤/飞手/财务
3. 10节点订单流转
4. 关键超时规则：
   - 接单 2 小时超时自动关闭
   - 用户确认报价 24 小时超时自动关闭
5. 议价最多 3 轮
6. 关键通知模板：报价、出发、完成、开票、关闭、投诉结果
7. 服务商移动端关键节点操作覆盖：
   - 前往吊运地点
   - 开始吊运
   - 吊运结束
   - 开具发票与收款记录

## 测试

执行：

```bash
npm run test:mini-program
```

测试会校验：
- 节点数量与顺序
- 关键角色节点权限
- 超时规则
- 页面门户完整性
- 消息模板覆盖
- 议价轮次上限

## 给其他模块的接入建议

- Web 端与小程序端共用 `types.ts` 和 `constants.ts` 中的状态字典
- 后端接口联调时，以 `fieldDictionary.ts` 作为前端字段约束基线
- 流程按钮显示逻辑统一调用 `flow.ts` 中权限与节点工具函数
