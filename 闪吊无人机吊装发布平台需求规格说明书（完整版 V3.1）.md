**1. 项目概述**

1.1 项目背景

随着无人机技术的快速发展和应用场景的不断拓展，无人机吊装服务在建筑工地、电力设备维护、光伏风电安装、应急物资配送等领域展现出巨大的市场需求。传统的吊装方式受限于地形、环境等因素，而无人机吊装以其灵活性、高效性和安全性成为新兴的解决方案。

闪吊无人机吊装发布平台旨在搭建一个连接服务需求方和服务提供方的撮合平台，通过整合无人机吊装服务商资源，为个人用户和企业商户提供便捷、高效的无人机吊装服务下单渠道。

1.2 项目目标

搭建统一的无人机吊装服务发布与接单平台

实现服务需求的在线发布、订单全流程跟踪、服务评价全流程

为入驻服务商提供专业的管理后台，支持员工按角色协作，提升运营效率

为终端用户提供网页端和小程序端双入口，随时随地发布需求，并通过消息通知实时了解订单节点状态

平台不介入任何收付款环节，仅向入驻服务商收取固定的软件服务年费 5000 元/年，不对用户端收取任何费用

1.3 目标用户

| 用户类型                          | 描述                             | 使用场景                               |
| --------------------------------- | -------------------------------- | -------------------------------------- |
| 个人用户                          | 有临时吊装需求的个人             | 小型物资搬运、紧急物资配送             |
| 企业用户                          | 建筑工地、电力公司、光伏风电企业 | 设备吊装、物资配送、应急响应           |
| 服务商（公司）                    | 无人机吊装服务商                 | 接单、管理员工、资产、财务、展示实力   |
| 服务商员工（客服/地勤/飞手/财务） | 服务商公司内部人员               | 根据角色权限处理订单对应节点           |
| 平台运营方                        | 平台管理员                       | 系统管理、商户审核、数据分析、投诉处理 |

1.4 运营模式

平台不触碰任何交易资金：所有吊装费用由用户与服务商线下自行结算（现金、对公转账等）。

平台收入来源：仅向入驻服务商收取固定的软件服务年费 5000 元/年，不按订单抽成，不向用户端收取任何费用。

服务商年费提供以下价值：订单线索（区域自动匹配）、订单全流程管理（含节点跟踪、消息通知）、员工与权限管理、固定资产管理、公司资料管理、服务商推广页面、数据看板、绩效报表等。

**2. 系统总体架构**

2.1 系统组成

系统由三大核心端组成：

| 端       | 子端                    | 访问方式            | 说明                                                         |
| -------- | ----------------------- | ------------------- | ------------------------------------------------------------ |
| 用户端   | 网页端 + 小程序端       | PC浏览器/微信小程序 | 个人/企业用户发布需求、查看订单、跟踪节点、评价、投诉        |
| 服务商端 | 网页端 + 小程序端       | PC浏览器/微信小程序 | 服务商及其员工（按角色登录）管理订单、员工、资产、财务等，小程序端功能与网页端对齐，支持移动场景操作 |
| 管理后台 | 网页端（独立域名/路径） | PC浏览器            | 平台运营方专用，不与服务商端共用登录入口，用于全局数据查看、商户审核、投诉处理等 |

两端数据互通，共享同一后端API。管理后台独立部署，使用不同域名或子域名，且不与服务商端共用登录态。

2.2 技术架构

| 小程序端   | 微信小程序原生开发      | 用户端、服务商端共用小程序，服务商端小程序必须支持所有节点操作（拍照上传、点击出发/吊运等） |
| 后端框架   | Spring Boot             | RESTful API 服务                                             |

| 消息推送   | 微信订阅消息 + 短信网关 | 向用户推送订单状态更新、报价提醒、评价邀请等                 |

JWT Token / OAuth2.0 身份认证

HTTPS 传输，敏感数据脱敏

操作审计日志

**3. 管理后台功能需求（面向平台运营方）**

管理后台为独立系统，使用单独域名（如 [admin.shandiao.com](https://admin.shandiao.com/)）或路径，不与服务商端共用登录。

3.1 系统管理模块

运营人员账号 CRUD，角色权限（超级管理员、运营管理员）

登录日志、操作审计

3.2 服务商管理

服务商入驻申请审核（营业执照、服务资质、保险证明等）

服务商信息查看与编辑

服务商禁用/启用控制

服务商区域设置：为每个服务商配置负责的区域（省/市/区县），支持一个服务商覆盖多个相邻地市

查看服务商经营数据（订单量、营收、评分、响应速度）

3.3 订单管理（全局）

全平台订单列表（多条件筛选：状态、服务商、时间、节点）

订单详情查看（含完整节点时间线）

异常订单处理（标记、备注）

3.4 投诉工单管理

用户发起的投诉工单列表（原因：服务态度、货损、乱收费等）

工单状态：待处理 → 处理中 → 已解决

平台运营方处理并记录处理结果，可向服务商下发整改建议

用户端投诉入口：在订单详情页“评价”下方增加“投诉”按钮，用户可提交投诉内容、凭证图片，工单自动关联订单。

3.5 数据汇总报表

全平台订单数据报表（订单量、成交额、取消率）

营收数据报表（仅平台服务费收入）

用户数据报表（新增用户、活跃用户、发布频次）

服务商数据报表（入驻数量、服务质量、接单率、区域覆盖）

自定义报表导出（Excel/CSV）

3.6 用户信息管理

注册用户列表，实名认证管理

用户黑名单管理

3.7 推广管理

广告位管理（首页 Banner、推荐位）

消息模板管理（用于短信、微信订阅消息）

3.8 活动策略管理

优惠券/红包配置（仅用于平台拉新，不涉及资金）

限时折扣活动、新用户礼包等

**4. 用户端功能需求（网页端 + 小程序端）**

面向个人用户和企业用户。

4.1 用户类型与登录

个人用户：手机号注册/微信授权登录（小程序）或手机号注册（网页）。

 

 

企业用户：需上传营业执照认证（可选），认证后可享受企业专属服务（如批量发布需求、专属客服等）。

4.2 首页模块

快捷发布需求按钮

附近服务商推荐（基于地理位置，按距离升序，支持排序切换）

Banner 轮播（活动推广）

4.3 发布吊装需求

所有字段均为必填（除“备注说明”外），水平距离和垂直距离系统自动计算+用户手动修正。

| 字段名称           | 类型                     | 说明                                    |
| ------------------ | ------------------------ | --------------------------------------- |
| 起始点地址         | 地图选点/文字输入        | 吊装物资取货位置                        |
| 送达点地址         | 地图选点/文字输入        | 吊装物资配送目的地                      |
| 水平距离           | 只读 + 可手动修改        | 系统自动计算（米/公里），用户可微调修正 |
| 垂直距离           | 数字输入（默认自动填充） | 系统根据高程自动估算（米），可手动修改  |
| 物资名称           | 文字输入                 | 具体名称                                |
| 总重量（kg）       | 数字输入                 |                                         |
| 单件最大质量（kg） | 数字输入                 |                                         |
| 体积（长宽高 cm）  | 数字输入                 | 长×宽×高                                |
| 期望日期           | 日期选择                 |                                         |
| 联系人             | 文本输入                 |                                         |
| 联系电话           | 手机号                   |                                         |
| 备注说明           | 多行文本（选填）         | 特殊要求                                |

提交后，系统根据起吊点地址自动匹配服务商（见5.5节），订单进入“待接单”状态，并生成唯一订单号。同时，系统通过微信订阅消息或短信向用户发送“需求已提交，等待服务商接单”的通知。

4.4 订单中心（增强）

用户可在订单中心查看自己发布的所有订单，列表展示订单号、起终点、状态、当前节点、操作按钮。

4.4.1 订单列表展示

订单状态标签：进行中、已完成、已关闭

当前节点（如“待服务商接单”、“等待用户确认报价”、“作业中”等）

点击订单进入详情

4.4.2 订单详情（节点时间线）

订单详情页以时间线（步骤条）形式展示完整节点及每个节点的操作人、操作时间、备注：

提交需求 → 服务商接单 → 服务商报价 → 用户确认报价 → 资源分配 → 前往吊运地点 → 开始吊运 → 吊运结束 → 开具发票 → 评价服务

每个节点完成后自动记录时间，已完成的节点高亮。

当前节点等待用户操作时（如“等待用户确认报价”、“待评价”），显示操作按钮。

节点状态通过微信订阅消息/短信实时推送：例如“服务商已报价，请确认”、“飞手已出发”、“吊运已完成，请评价”等。

4.4.3 用户操作

确认报价：服务商报价后，用户可点击“确认报价”进入下一节点。若用户对报价不满意，可点击“拒绝报价”并填写期望价格（或说明），服务商可修改报价重新提交（最多3轮议价），若超时（24小时）未达成一致，订单自动关闭。

评价服务：吊运结束后，用户可对本次服务进行1-5星评价，并填写评语。

查看关闭原因：若服务商因故关闭订单（如无法提供服务），需填写理由，用户可在订单详情中查看。

发起投诉：订单完成后，用户可在订单详情页“评价”下方点击“投诉”，选择投诉原因（服务态度差、货损、乱收费等），上传凭证，提交后生成工单至管理后台。

4.4.4 服务商同步信息

用户端可实时看到服务商侧更新的节点信息，例如“服务商已分配飞手”、“飞手已出发前往吊运地点”、“飞手开始吊运”等，每个节点都有对应状态，并通过消息通知告知用户。

4.5 我的模块

个人信息管理（头像、昵称、手机号）

地址簿管理（常用地址收藏）

联系客服

投诉记录（查看自己提交的投诉工单状态）

**5. 服务商端功能需求（网页端 + 小程序端）**

面向服务商及其员工，根据角色权限展示不同功能和数据。服务商端小程序必须完整支持所有节点操作（如点击“前往吊运地点”、“开始吊运”、“开具发票”等），并支持拍照上传、图片选择等移动端特性。

5.1 区域自动匹配服务商规则

平台运营方在管理后台为每个服务商配置负责区域（省/市/区县，支持多选，一个地市或相邻地市可由一个服务商覆盖）。

用户发布需求时，系统根据起吊点地址（行政区划）自动匹配负责该区域的服务商，订单直接推送至该服务商。

若起吊点地址无法精确匹配，则按就近原则匹配至相邻区域服务商（由后台配置的默认规则决定）。

匹配规则支持后续扩展（如按距离、按服务商负载）。

 

5.2 登录与角色

服务商管理员在平台入驻审核通过后，获得管理员账号。

管理员可登录服务商端（网页/小程序），并创建员工账号，分配角色。

员工角色与对应操作：

| 角色   | 可操作节点                            | 权限范围                                        |
| ------ | ------------------------------------- | ----------------------------------------------- |
| 客服   | 接单、报价、资源分配（指派飞手/地勤） | 查看所有订单（不可修改财务）                    |
| 地勤   | 前往吊运地点                          | 仅查看被分配的任务订单，更新该节点              |
| 飞手   | 开始吊运、吊运结束                    | 仅查看被分配的订单，更新该节点，可上传照片/视频 |
| 财务   | 开具发票、标记收款                    | 查看已完成订单，开具发票并记录收款信息          |
| 管理员 | 全部节点                              | 全部功能、全部数据                              |

不同角色登录后，菜单和数据范围自动过滤。

5.3 首页仪表盘

今日订单概览（待接单、进行中、已完成数量）

本月营收统计（仅统计已结算订单金额）

服务评分与最新评价

快捷操作入口（接单、新增资产、新增员工）

5.4 订单管理（全节点流程）

5.4.1 订单列表

支持按状态筛选：待接单、进行中、已完成、已关闭

支持按节点筛选（如“待报价”、“待资源分配”等）

每个订单显示当前节点和操作按钮

5.4.2 订单详情与节点操作

订单详情页以时间线形式展示所有节点，每个节点有明确的“操作”入口（根据角色权限显示）。

节点1：服务商接单（客服/管理员）

操作：点击“接单”，订单进入下一节点。若无法服务，需填写关闭理由（必填），订单状态变为“已关闭”，用户端可查看理由。

节点2：服务商报价（客服/管理员）

操作：填写报价金额（必填）和报价说明（选填），提交后订单进入“等待用户确认报价”状态。

若用户拒绝报价，服务商可修改报价再次提交（最多3轮），系统记录每次报价历史。

节点3：用户确认报价（用户端操作，服务商端只读）

服务商端可查看用户是否确认，确认后方可继续。若用户拒绝且议价超时（24小时无新报价），订单自动关闭。

节点4：资源分配（客服/管理员）

操作：从员工列表中选择飞手、地勤（必选），分配后订单进入下一节点。

节点5：前往吊运地点（地勤角色）

操作：地勤登录后，在订单详情点击“前往吊运地点”，记录出发时间（可选上传出发照片）。

节点6：开始吊运（飞手角色）

操作：飞手到达现场后，点击“开始吊运”，记录开始时间（可选上传开始作业照片）。

节点7：吊运结束（飞手角色）

操作：作业完成后，点击“吊运结束”，填写实际重量、实际体积、实际金额（用于服务商内部统计），上传作业凭证（照片/视频）。

节点8：开具发票（财务角色）

操作：财务人员点击“开具发票”，填写发票号码、开票时间，并标记收款（记录收款方式、金额、时间），订单进入“待评价”状态。

节点9：评价服务（用户端操作，服务商端只读）

服务商可查看用户评价。

5.4.3 订单超时自动处理

接单超时：订单推送后2小时内无人接单，系统自动关闭订单，并通知用户。

用户确认报价超时：服务商报价后24小时内用户未操作，订单自动关闭。

作业节点提醒：资源分配后，若飞手/地勤超过1小时未操作“前往吊运地点”，系统向服务商管理员发送提醒（站内信），不自动关闭。

超时规则可在管理后台配置。

5.5 员工管理（仅管理员可见）

员工列表：姓名、角色、手机号、累计执行订单数（自动统计）

增删改员工，设置角色

绩效看板：按角色展示绩效数据

飞手：完成订单数、好评率、平均作业时长（从节点时间自动计算），执照编号，照片

客服：接单数、报价响应时间（从接单到报价的平均时长），照片

地勤：出发准时率（按期望日期与实际出发时间的对比）。照片

数据可导出

5.6 公司资料与资质管理（仅管理员可见）

公司基本信息、简介、服务区域（由平台运营方在管理后台配置，服务商不可修改）

资质上传（运营许可证、飞手执照、保险单等）

安全飞行时长（手动录入或后期自动累加）

5.7 固定资产管理（仅管理员可见）

资产类型：无人机、车辆、吊具、其他

字段：类型、型号/名称、序列号/车牌号、购置日期、保险信息、保养日期、保险到期日期，照片，状态

增删改资产，保险/保养提醒

5.8 服务管理（仅管理员可见）

**6. 订单全节点流程总结**

| 节点            | 操作方            | 说明                                                 |
| --------------- | ----------------- | ---------------------------------------------------- |
| 1. 提交需求     | 用户              | 用户填写表单，系统自动匹配服务商，订单生成，发送通知 |
| 2. 服务商接单   | 服务商客服/管理员 | 接单或关闭（关闭需理由），超时自动关闭               |
| 3. 服务商报价   | 服务商客服/管理员 | 填写报价金额，用户确认前不可继续，支持最多3轮议价    |
| 4. 用户确认报价 | 用户              | 确认后订单进入下一节点，超时自动关闭                 |
| 5. 资源分配     | 服务商客服/管理员 | 指派飞手、地勤                                       |
| 6. 前往吊运地点 | 服务商地勤        | 地勤更新状态，记录出发时间                           |
| 7. 开始吊运     | 服务商飞手        | 飞手更新状态，记录开始时间                           |
| 8. 吊运结束     | 服务商飞手        | 飞手更新状态，填写实际数据，上传凭证                 |
| 9. 开具发票     | 服务商财务        | 财务更新状态，记录发票信息及收款                     |
| 10. 评价服务    | 用户              | 用户评价，订单完成。用户可发起投诉                   |

每个节点完成后，系统自动记录操作人、操作时间，并推送状态更新给用户（通过微信订阅消息/短信）。

用户端可看到完整的节点时间线，服务商端可看到所有节点记录。

**8. 商业模式补充说明**

平台不建立任何收付款系统，不托管资金，不参与交易。

用户端：永久免费发布需求。

服务商端：按年收取软件服务费 5000 元/年，费用通过线下对公转账或平台提供的支付接口缴纳（该接口仅用于收取服务费，不涉及订单交易）。

服务费提供的价值：订单线索（区域自动匹配）、订单全流程管理（含节点跟踪、消息通知）、员工与权限管理、固定资产管理、公司资料管理、服务商推广页面、数据看板、绩效报表等。

禁止：平台不得以任何理由向用户收取发布费、服务费，也不得从服务商的订单中抽成。

**9. 附录**

9.1 距离自动计算技术说明

水平距离：使用 Haversine 公式根据起终点经纬度计算大圆距离，单位米。前端地图 SDK（如腾讯地图、高德地图）提供测距接口。

垂直距离：通过地图高程 API（如腾讯地图海拔服务）获取起终点高程差。若 API 不可用或精度不足，则允许用户手动输入。

9.2 区域自动匹配实现说明

管理后台配置服务商负责区域（支持省/市/区县多级）。

用户发布需求时，解析起吊点地址的行政区划（通过地图逆地理编码），与配置区域匹配。

若同一区域有多个服务商，可暂按随机或轮询分配（后续可扩展按评分、距离等）。

9.3 消息推送技术说明

小程序端：使用微信订阅消息，需用户授权一次，长期有效。

短信：接入第三方短信服务商（如阿里云、腾讯云），仅用于关键节点，控制成本。

站内信：存入数据库，用户登录网页端时展示。
